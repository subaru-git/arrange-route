# 仕様書：ダーツ「アレンジ辞書」SNS（日本向け）

## 背景・目的

ダーツにおけるアレンジはパターンが有限であるが、優先度には流行り廃りがある。
また、アレンジ表は存在するが、利点や外れた時のルートなどが明確でなく、これらを有識者から口頭で聞き、議論するのが業界のスタンダードと言える。
そこで、このプロダクトでは、アレンジのルートを簡単に登録でき、VOTEによって王道や流行りがわかるようにすることで、プレイヤーを補助する。

### ゴール

- 残り点数（例：70）ごとに、1本/2本/3本のアレンジ投稿が集まる
- 各種アウトルール(Double Out / Master Out / Single Out)ごとにルートが明確である
- 「ミス時分岐」が構造化され、議論と投票で上位が育つ
- 共有URLで拡散できる（/scores/70）
- VOTEとコメントができる

### 非ゴール（MVPではやらない）

- フォロー/タイムライン中心のSNS化
- 画像・動画投稿
- 複雑な全文検索、レコメンドAI

## 想定ユーザー（ペルソナ）

- 中級者：RT8〜12くらい。「上がり目を逃したくない」。外れた時を含めた最適解が知りたい
- 上級者：RT12〜。**「成功率・期待値」「リスク管理」「勝ち筋」**を議論したい。
- 初級者：まずは閲覧メイン。ログインせずに使いたい。

## コアコンセプト（辞書型）

- アプリの入口は常に 点数一覧ページ
- 入口URLは `/scores` とする
- 投稿は「自由文章」ではなく、最低限の構造化（後から検索・比較できるため）
- 共有されるのは投稿ではなく 点数ページが基本

## MVPスコープ（機能要件）

### 閲覧（ログイン不要）

#### 点数ページ（主役）

- URL：/scores/{remaining_score}?out_rule=...&bull_mode=...
- フィルター：out_rule（Double Out / Master Out / Single Out）, bull_mode（セパレート / ファット）
- 検索パラメータ未指定時は全表示とする
- 表示：投稿カード一覧（vote_score desc, created_at desc / 新着順）
- 共有：URLコピーボタン

### 投稿（ログイン必須）

#### 投稿作成

- URL：/new
- remaining_score（残り点数: 1-701）, 残り本数 1-3 を選択
- out_rule（Double Out / Master Out / Single Out）を選択
- bull_mode（セパレート / ファット）を選択
- 初手の狙いを入力 (i.e, T20)
- 一般的なアレンジ記法は未統一である前提とし、本プロダクト内でのみトークン表記を正規化する
- トークン表記は `bull_mode=ファット` の場合は `BULL(50)` のみとし、`bull_mode=セパレート` かつ `out_rule=Double Out` の場合はPDC準拠、それ以外は日本ダーツ協会準拠とする（例：BULL表記はセパレート+Double Outでは `Bull/25`、それ以外のセパレートでは `S-BULL/D-BULL`）
- 成功ルートを入力(i.e, T20 → D5)
- ミス分岐を入力。入力は初手の狙いの周辺のみ（例：T20狙いなら、S20/T5/S5/T1/S1）
- 分岐の途中からでも分岐を作成可能(S20 → S20 -> DBULL と入力済みの場合、S20からの分岐に S18 → D16を追加可能)
- 成功ルート/ミス分岐の各ルートは、残り本数に達した時点、または残り点数が0になった時点で終了すること
- 入力はコメント以外全てボタンによる選択式（一般的な記法が未統一なため）
- ルート入力UIは、右に行くほど展開するTree Viewで行う（連結記号の直接入力は不要）
- 基本フローは分岐のボタン入力のみで完結し、コメント入力は任意（理由を補足したい場合のみ）とする
- 保存後：/scores/{remaining_score} に遷移
- 投稿の編集は投稿者本人のみ可能

#### 投票（↑/↓）

- 未ログインユーザーも投票可能
- 未ログインユーザーの同一判定はブラウザID（Cookie/LocalStorage）で行う
- 未ログインユーザーのブラウザID有効期間は365日とする
- upvote / downvote の両方を提供する
- 票は取り消し可能
- vote_score = upvote数 - downvote数
- 点数ページの人気順は vote_score desc, created_at desc で並べる
- 不正投票対策（連投検知・自動reversal等）はMVPでは除外
- IP単位の不正利用対策は実施しない（MVP）

#### コメント

- 1投稿に対してコメント追加
- 削除は本人のみ（MVP）

### アカウント（最小）

#### ログイン/ログアウト

- Social Login(Google)のみ

### 画面要件（UI/UX）

#### 点数ページ（/scores/[remaining_score]）

- ヘッダ：{remaining_score} out option / 残り本数
- サブ：投稿数
- CTA：この点数に投稿する → /new

#### 投稿カード（一覧で必ず見える要素）

- Main Route（太字）
- Miss Route （成功ルートから分岐している場合のみ、矢印で繋いで表示）
- vote_score（または↑/↓内訳）、コメント数
- 投稿者（display_name）、投稿日時（相対表記。日時はUTCで保存し、表示はユーザー環境のローカル時刻で行う）

#### 投稿作成（/new）

- Route入力
- Miss Plan
- 右方向に展開するTree Viewでルートを編集
- コメント入力（任意）

## 技術スタック

- フロントエンド：Next.js
- ホスティング：Vercel
- BaaS/DB/Auth：Supabase
- MVP方針：技術スタック全体を無料枠/無料プランで運用する（Domain費用は別）
- 将来的な置き換え：Supabase依存を前提にせず、AWS等への移行可能性を考慮した構成にする

## 受け入れ条件（Acceptance Criteria）

### MVPリリース条件

- /scores/70?out_rule=double_out&bull_mode=separate を開くと、条件に一致する投稿一覧が表示され、vote_score desc, created_at desc / 新着順に切替できる
- /scores/70 では out_rule と bull_mode のフィルターを操作すると検索パラメータ（`?out_rule=...&bull_mode=...`）に反映される
- /scores/70（検索パラメータ未指定）では全表示される
- フィルターが未指定状態に戻った場合は、該当検索パラメータをURLから削除する
- 未ログインでも閲覧できる
- ログインすると投稿作成ができ、投稿が点数ページに反映される
- 投稿作成時に out_rule と bull_mode を選択できる
- 投稿作成画面で、ルート入力が右方向Tree Viewかつコメント以外はボタン選択で入力できる
- 未ログインを含めて↑/↓投票と取り消しができ、vote_score desc に影響する
- コメントが追加でき、/scores/{remaining_score} の投稿カード上で表示される
- コメントは投稿者本人のみ削除できる
- 投稿は投稿者本人のみ編集できる
- 投稿/投票/コメントの権限がRLSで守られている

## リリース方針

- 正式リリースは、独自ドメイン設定の完了をもって実施する
